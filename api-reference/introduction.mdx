## Overview

The Surf Data REST API allows you to programmatically manage projects, datasources, tools, and tokens. All endpoints are authenticated using Clerk session tokens.

## Base URL

```
https://app.surfdata.io/api
```

## Authentication

All API requests require authentication via Clerk session cookies. When making requests from the browser (dashboard), authentication is handled automatically. For programmatic access, include the session token in the request headers.

```bash
curl https://app.surfdata.io/api/projects \
  -H "Cookie: __session=your-clerk-session-token"
```

<Note>
  The REST API is designed for use with the Surf Data dashboard and authenticated sessions. For agent-to-database communication, use the [MCP endpoint](/api-reference/mcp-endpoint) with Bearer tokens.
</Note>

## Response format

All responses use JSON format.

### Success response

```json
{
  "data": { ... }
}
```

### Error response

```json
{
  "error": "Error message describing what went wrong"
}
```

### HTTP status codes

| Code | Description |
|------|-------------|
| `200` | Success |
| `201` | Created |
| `400` | Bad request — invalid parameters |
| `401` | Unauthorized — missing or invalid authentication |
| `403` | Forbidden — insufficient permissions |
| `404` | Not found — resource doesn't exist |
| `409` | Conflict — resource limit reached |
| `500` | Internal server error |

## Rate limits

The REST API does not currently enforce rate limits beyond your plan's execution limits for tool invocations. However, we recommend:

- Avoid polling endpoints more than once per second
- Use webhooks for real-time updates when available
- Cache responses where appropriate
