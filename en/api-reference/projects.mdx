## List projects

Returns all projects for the authenticated organization.

```
GET /api/projects
```

### Response

```json
{
  "projects": [
    {
      "id": "clx1234567890",
      "name": "Customer Support Agent",
      "description": "Tools for customer lookup and order tracking",
      "organizationId": "org_123",
      "createdById": "user_456",
      "isPublished": true,
      "createdAt": "2024-01-15T10:30:00Z",
      "updatedAt": "2024-01-20T14:00:00Z",
      "_count": {
        "datasources": 2,
        "agentViews": 5,
        "mcpTools": 8
      }
    }
  ]
}
```

<ResponseField name="projects" type="object[]" required>
  Array of project objects.
  <Expandable>
    <ResponseField name="id" type="string" required>Unique project identifier</ResponseField>
    <ResponseField name="name" type="string" required>Project name</ResponseField>
    <ResponseField name="description" type="string">Project description</ResponseField>
    <ResponseField name="organizationId" type="string" required>Owner organization ID</ResponseField>
    <ResponseField name="isPublished" type="boolean" required>Whether the project is published</ResponseField>
    <ResponseField name="createdAt" type="string" required>ISO 8601 creation timestamp</ResponseField>
    <ResponseField name="updatedAt" type="string" required>ISO 8601 last update timestamp</ResponseField>
    <ResponseField name="_count" type="object">
      Resource counts.
      <Expandable>
        <ResponseField name="datasources" type="number">Number of datasources</ResponseField>
        <ResponseField name="agentViews" type="number">Number of agent views</ResponseField>
        <ResponseField name="mcpTools" type="number">Number of MCP tools</ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

## Create project

Creates a new project in the authenticated organization.

```
POST /api/projects
```

### Request body

<ParamField body="name" type="string" required>
  The project name. Must be non-empty.
</ParamField>

<ParamField body="description" type="string">
  Optional project description.
</ParamField>

### Example

```bash
curl -X POST https://surfdata.com.br/api/projects \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Sales Analytics Bot",
    "description": "Tools for revenue and pipeline queries"
  }'
```

### Response

```json
{
  "id": "clx1234567890",
  "name": "Sales Analytics Bot",
  "description": "Tools for revenue and pipeline queries",
  "organizationId": "org_123",
  "createdById": "user_456",
  "isPublished": false,
  "createdAt": "2024-01-15T10:30:00Z",
  "updatedAt": "2024-01-15T10:30:00Z"
}
```

## Get project

Returns a single project by ID.

```
GET /api/projects/{id}
```

<ParamField path="id" type="string" required>
  The project ID.
</ParamField>

### Response

Returns the project object with full details including relations.

## Update project

Updates a project's name or description.

```
PUT /api/projects/{id}
```

<ParamField path="id" type="string" required>
  The project ID.
</ParamField>

<ParamField body="name" type="string">
  New project name.
</ParamField>

<ParamField body="description" type="string">
  New project description.
</ParamField>

---

## Delete project

Permanently deletes a project and all its resources (datasources, views, tools, tokens).

```
DELETE /api/projects/{id}
```

<ParamField path="id" type="string" required>
  The project ID.
</ParamField>

<Warning>
  This action is irreversible. All datasources, views, tools, and tokens associated with the project will be permanently deleted.
</Warning>
